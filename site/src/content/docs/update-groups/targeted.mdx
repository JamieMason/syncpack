Clamp the highest registry update offered for matched instances. The CLI's [`--target`](/command/update/#--target) value still applies — the stricter of the two wins — so a group with `target: "latest"` cannot loosen a CLI run that used `--target patch`.

## Configuration

### target <Badge text="Required" variant="danger" />

One of `"patch"`, `"minor"`, or `"latest"`.

```json title=".syncpackrc.json"
{
  "updateGroups": [
    {
      "dependencies": ["storybook", "@storybook/**"],
      "target": "patch"
    },
    {
      "dependencyTypes": ["dev"],
      "target": "minor"
    }
  ]
}
```

| Value    | Allowed update relative to current version |
| -------- | ------------------------------------------ |
| `patch`  | Same major and minor; higher patch only.   |
| `minor`  | Same major; higher minor or patch.         |
| `latest` | Any newer version, including major bumps.  |
