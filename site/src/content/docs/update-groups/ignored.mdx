Skip these dependencies entirely when looking for registry updates. Matched instances are reported as valid and are not offered as candidates by the [update](COMMAND_UPDATE) command, even when newer versions exist on the npm registry.

A common use is opting peer dependencies out of updates, since the consuming project — not the publisher — owns the peer range.

## Configuration

### isIgnored <Badge text="Required" variant="danger" />

This property activates this behaviour for a given Update Group.

```json title=".syncpackrc.json"
{
  "updateGroups": [
    {
      "dependencyTypes": ["peer"],
      "isIgnored": true
    }
  ]
}
```
